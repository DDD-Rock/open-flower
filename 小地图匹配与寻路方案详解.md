## 🎮 移动能力说明

移动能力基于 `automation/human_input.py` 中的 `HumanInput` 类实现。

### 拟人化设计原理

所有按键操作都经过拟人化处理，避免机器人特征：

1. **按键时长随机化** - 使用正态分布生成随机按住时长
2. **按键间隔随机化** - 组合键之间有随机延迟
3. **偶尔停顿** - 5%概率产生短暂间歇
4. **换向延迟** - 先松开一个方向键，延迟后再按另一个

### 核心时间参数（基于真人录制数据）

```python
# 方向键参数
direction_tap_duration = (40, 120)      # 轻点微调(ms)
direction_change_delay = (50, 200)      # 换向延迟(ms)

# 传送门按键（上键）
portal_press_duration = (200, 800)      # 按住时长(ms)

# 组合键偏移
key_offset_range = (10, 80)             # 两键不同时按的偏移(ms)
```

### 拟人化随机时长生成

```python
def _random_duration(self, range_ms: Tuple[int, int]) -> float:
    """使用正态分布生成随机时长，大多数值接近中间"""
    min_ms, max_ms = range_ms
    mean = (min_ms + max_ms) / 2
    std = (max_ms - min_ms) / 4
    duration = random.gauss(mean, std)
    duration = max(min_ms, min(max_ms, duration))
    return duration / 1000.0  # 转为秒
```

## 🔧 核心移动 API

### 1. 开始移动

```python
from automation.human_input import HumanInput

human = HumanInput()

# 向左移动（按住左键）
human.move_left()

# 向右移动（按住右键）
human.move_right()

# 停止移动（松开方向键）
human.stop_move()
```

**拟人化细节**：
- 换向时先松开当前方向键，延迟 50~200ms 后再按新方向键
- 不会同时按住左右两个方向键

### 2. 使用传送门

```python
# 到达传送点后按上键进入传送门
human.use_portal()
```

**拟人化细节**：
- 按住上键 200~800ms（正态分布随机）

### 3. 轻点微调

```python
# 用于精确定位时的轻点
human.tap_direction('left')   # 轻点左
human.tap_direction('right')  # 轻点右
```

**拟人化细节**：
- 按住时长 40~120ms

## 🎯 完整使用流程：移动到传送点

以下代码展示如何移动到 shichang 的传送点并进入传送门：

```python
import json
import time
import cv2
import win32gui
from detection.minimap_monitor import MinimapMonitor
from automation.human_input import HumanInput

# ========== 1. 加载配置 ==========
with open("templates/maps/shichang_config.json", "r", encoding="utf-8") as f:
    config = json.load(f)

# ========== 2. 初始化 ==========
monitor = MinimapMonitor()
hwnd = win32gui.FindWindow(None, "MapleStory")
monitor.set_window_handle(hwnd)
monitor.load_full_map(
    full_map_path=config["full_map_path"],
    original_minimap_size=tuple(config["full_map_original_minimap_size"])
)
monitor.auto_detect_dark_region()

human = HumanInput()

# ========== 3. 获取地图尺寸 ==========
full_map = cv2.imread(config["full_map_path"])
full_map_height, full_map_width = full_map.shape[:2]

# ========== 4. 设置目标传送点 ==========
target = config["teleport_points"]["portal_1"]
target_abs_x = target["rel_x"] * full_map_width

# ========== 5. 导航到传送点 ==========
TOLERANCE = 3  # X坐标容差范围 ±3像素

while True:
    result = monitor.find_player_position_on_full_map()
    if not result:
        time.sleep(0.05)
        continue
    
    current_x, current_y, confidence = result
    dx = target_abs_x - current_x
    
    # 判断是否到达目标（X坐标±3像素范围内）
    if abs(dx) <= TOLERANCE:
        # 到达传送点，停止移动
        human.stop_move()
        time.sleep(0.1)  # 短暂等待稳定
        
        # 按上键进入传送门
        human.use_portal()
        print("已进入传送门")
        break
    
    # 根据方向移动
    if dx > TOLERANCE:
        human.move_right()
    elif dx < -TOLERANCE:
        human.move_left()
    
    # 定位刷新频率
    time.sleep(0.05)
```

## 📐 坐标转换

配置文件中存储的是**相对坐标**（0~1），使用时需转换为**绝对坐标**（像素）：

```python
# 相对坐标 → 绝对坐标
abs_x = rel_x * full_map_width
abs_y = rel_y * full_map_height
```

---

## 💡 注意事项

1. **容差范围**：传送点判断使用 X 坐标 ±3 像素作为到达判定
2. **刷新频率**：定位刷新间隔建议 50ms
3. **拟人化**：所有按键都经过随机化处理，无需额外处理
4. **安全释放**：异常退出时调用 `human.release_all()` 释放所有按键

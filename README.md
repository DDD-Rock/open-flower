# 冒险岛世界祭司自动技能释放工具

## 项目简介

这是一个用于控制冒险岛世界游戏中祭司职业自动释放技能的Python工具。通过图形化界面，您可以设置多个技能按键及其释放间隔，工具会自动按照设定的时间间隔和随机延迟来释放技能。

## 功能特性

### 当前功能
- ✅ 图形化UI界面，操作简单直观
- ✅ 支持设置多个技能按键
- ✅ 支持设置每个技能的释放间隔（秒）
- ✅ 支持设置随机延迟秒数（增加随机性，避免被检测）
- ✅ 自动计时和循环释放技能
- ✅ 一键启动/停止功能

### 未来拓展功能（已预留接口）
- 🔜 小地图识别功能
- 🔜 自由市场按钮识别功能
- 🔜 技能释放完成后自动进入小地图功能

### 操作步骤

1. **添加技能**：
   - 在"技能按键"输入框中输入技能对应的按键（如：1, 2, 3, F1, F2等）
   - 在"释放间隔（秒）"输入框中输入该技能的释放间隔时间
   - 在"随机延迟（秒）"输入框中输入随机延迟的最大秒数
   - 点击"添加技能"按钮

2. **管理技能列表**：
   - 在技能列表中可以看到所有已添加的技能
   - 可以选中某个技能后点击"删除技能"按钮移除

3. **启动/停止**：
   - 点击"开始运行"按钮开始自动释放技能
   - 点击"停止运行"按钮停止自动释放
   - 运行状态会在界面上显示

## 技术说明

### 技能释放逻辑

1. 程序会按照技能列表的顺序依次释放技能
2. 每个技能释放后，会等待：**释放间隔 + 随机延迟（0到设定值之间）** 秒
3. 所有技能释放完成后，会重置计时，重新开始循环

### 按键模拟

程序使用 `keyboard` 库来模拟按键操作，确保按键操作准确可靠。

### 随机延迟说明

随机延迟是为了增加操作的随机性，避免被游戏系统检测为自动化脚本。每次释放技能后，会在0到您设定的最大随机延迟秒数之间随机选择一个值。

## 项目结构

```
open_flower/
├── README.md                 # 项目说明文档
├── requirements.txt          # Python依赖包列表
├── main.py                   # 程序入口文件
│
├── models/                   # 数据模型模块
│   ├── __init__.py
│   └── skill_config.py      # 技能配置数据模型
│
├── workers/                  # 工作线程模块
│   ├── __init__.py
│   └── skill_worker.py      # 技能释放工作线程
│
├── ui/                      # UI界面模块
│   ├── __init__.py
│   └── main_window.py       # 主窗口界面
│
├── utils/                   # 工具函数模块
│   ├── __init__.py
│   └── keyboard_utils.py    # 键盘操作工具函数
│
└── config/                  # 配置模块
    └── __init__.py          # 应用配置和常量定义
```

### 架构说明

项目采用模块化设计，各模块职责清晰，便于维护和扩展：

- **models/**: 数据模型层，定义数据结构（如技能配置）
- **workers/**: 业务逻辑层，处理后台任务（如技能释放循环）
- **ui/**: 用户界面层，负责界面显示和用户交互
- **utils/**: 工具函数层，提供通用工具函数（如按键模拟）
- **config/**: 配置层，集中管理应用配置和常量

这种架构设计使得：
- 代码结构清晰，易于理解和维护
- 各模块独立，便于单独测试和修改
- 方便未来扩展新功能（如图像识别模块）

## 打包成 EXE

### 安装 PyInstaller
```bash
pip install pyinstaller
```

### 打包命令
```bash
pyinstaller build.spec
```

或使用一行命令：
```bash
pyinstaller --onefile --noconsole --name "苗圃助手" --add-data "templates;templates" --add-data "settings.ini;." main.py
```

### 输出位置
打包完成后，EXE 文件位于 `dist/苗圃助手.exe`

> **注意**: 目标机器可能需要安装 Visual C++ Redistributable 才能运行
